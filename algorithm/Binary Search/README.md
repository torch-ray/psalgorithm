### 이분탐색
> 이분탐색(Binary Search) 이란?
- 이분탐색 또는 이진탐색은 순차 탐색과 마찬가지로 탐색의 한 가지 방법
- 이분탐색의 경우는 모든 데이터를 탐색하지 않으며 값의 크기로 추측하여 탐색을 진행
- 따라서 전제 조건으로 데이터가 정렬되어 있는 경우에만 가능
- 즉, 이미 정렬되어 있는 데이터의 범위를 반씩 줄여가며 탐색하는 알고리즘

### 구현
> 정렬된 데이터이기 때문에 특정 값을 기준으로 원하는 값의 위치추측 가능
```
[1, 4, 5, 7, 9, 14, 16, 18, 20, 30, 41]
```

- 위와 같은 배열에서 20의 위치를 찾아야하는 경우
- 순차 탐색의 경우에는 0번째 인덱스부터 원하는 값이 나올때 까지 순서대로 탐색
- 이분 탐색의 경우 가운데 값을 구하고 ( 이 경우 11개 값의 가운데 인덱스는 5이기 때문에 14 )
- 가운데 값과 원하는 값을 비교하여 원하는 값이 더 큰 경우 가운데 값의 오른쪽을 탐색, 원하는 값이 더 작은 경우에는 가운데 값의 왼쪽을 탐색하고 원하는 값과 같을 경우에는 해당 값의 인덱스를 리턴
- 20은 14보다 크기 때문에 이번엔 16부터 41사이의 가운데 값을 지정(20)
- 다시 한 번 원하는 값과 비교 ( 이 경우 원하는 값과 일치하기 때문에 바로 인덱스를 리턴 )
- 순차탐색으로 8번만에 찾을 수 있는 값을 이분탐색을 활용하여 2번만에 탐색완료
- 배열 데이터의 값이 많을수록 이분탐색의 탐색시간이 더 빠름

### 풀이코드
- 이분탐색의 경우 가운데 값을 기준으로 탐색 범위를 반으로 줄여나가기 때문에 O(logN)의 시간복잡도를 가짐
- 해당 문제와 같이 탐색해야 하는 값이 많은 경우 매우 유용하게 사용 가능
```swift
func solution(_ n:Int, _ times:[Int]) -> Int64 {
    var (left, right) = (times.min()!, times.max()! * n)
    var mid = 0
    var res = 0
    
    while left <= right {
        mid = (left+right)/2
        let availableCount = times.reduce(0) { $0 + mid/$1 }
        
        if availableCount >= n {
            res = mid
            right = mid-1
        } else {
            left = mid+1
        }
    }
    
    return Int64(res)
}
```
